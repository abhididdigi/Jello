Print and Evaluate functions:

print(with a small `p` only) function:

We all know that Jelly is a templating language. We mix server side constructs with HTML that gives us ability to write easy template pages. What ever you write in the print function, will be the HTML that will be converted. print function can take any number of arguments. For example: The Javascript code : print(" Hello world"); will be convered to : Hello World
If there is no quotes between the words, Jello will interpret it as a Jelly variable. For example: print(a); will be converted to: ${jvar_a}
Now, let's mix and match: print("Hello world", a);
will be rendered to Hello World${jvar_a}

evaluate(with a small `e` only) function:

Let's start by an example:

  • Declaration of the script:
    In this step, we handle the Server side Javascript code declaration such as Glide, Javascript array declaration, etc. here is an example:

    Say you want your g:evaluate loop to have a glide to Incident table

    Here is how you write it in Jello:

    var incGlideServerCode = 'var gr = new GlideRecord("incident"); gr.get("sys_id"); gr';
    evaluate(incGlideServerCode, "incidentGlide", "phase1");

    will be converted to:

Things to note here :

1) The declaration ( here var incGlideServerCode) should always have the word ServerCode in it. That's how Jello will identify that the value of this variable is going to be a server side javascript code.

2) evaluate function consists of three parameters.

one : The variable in which the Server side code is stored, here it is incGlideServerCode.

Two: The Jelly variable to which the results of Glide should be returned. Here it is incidentGlide. Hence it is convered to jvar_incidentGlide and used.

Three: For future reference, it also takes the phase of this evaluate.
