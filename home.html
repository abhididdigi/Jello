Jello

A small Converter to convert Javascript to Apache Jelly.

What is Jello?

Jello is a tiny Converter from Javascript to Apache Jelly. It takes an input in the form of Javascript, and converts this to Apache Jelly. Please visit the Examples section for demos.

Why Jello?

Apache Jelly is a scary language. At least it was for me when I first started. Most of my friends who start with Service Now also share the same opinion. Of course there are some excellent blogs out there to get you started. Jello is one step forward where in you can create your UIPages/Macros by writing Javascript code,without the knowledge of Apache Jelly syntax, just its concepts.

Okay,so how can I use it in my Service Now instance?

Right now, Jello is still in its infancy. There is a lot that will change. So until then, I suggest you use the editor in the examples page. But if you are really interested, you can export the Source of Jello into a UI Script, and use it.More information on dependencies section below.

Dependencies & Usage:

Jello uses Esprime Parser to parse the Javascript into tokens and Underscore for templating. Sounds scary, but it's very simple. The original idea was to create a separate Compiler with Jison which supports a simple language subset of Javascript, but I had to settle for Javascript as most of the folks working with Service Now know Javascript already, and I had to learn Jison(compilers all over again!)

Dependencies

There are two dependencies for Jello:
  • Esprime.js
  • Underscore.js
Usage

Download the Scripts from the above websites and install them as UI scripts in your service now instance.
Next install AstParser.js from Jello's Github repository and include all the three in a single UI page. Next, you can convert the Javascript code to Apache Jelly using the following code:
var finalJelly = AstParser.parse(esprima.parse("Javascript code goes here"));
finalJelly will now have the Apache jelly interpreted.

A note of caution, and road ahead

Before you start using Jello, here is what Jello can do, It can convert Javascript string into Apache Jelly. But the Apache Jelly will be in Phase one only. The conversion of Javascript into Phase two is still under active development. Jello yet doesn't have a robust Exceptional Handling mechanism. Even if something goes wrong, it still will be silent, until you figure out, something is wrong with your Javascript code. Things that are WIP:

  • Exceptional Handling
  • Support Jelly phase two
  • More control structures
  • Easy export to Service Now instances
with these caveats, you are now ready to use Jello, Have fun!
